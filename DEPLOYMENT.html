🚀 部署指南

将Retire-Cluster文档部署到GitHub Pages

📖 关于这个部署
这个指南将帮助你将Retire-Cluster文档部署到你的4ier.github.io GitHub Pages站点。文档将支持双语（中英文）并具有专业的响应式设计。

🎯 快速部署步骤

1 创建GitHub Pages仓库

首先，你需要创建一个GitHub Pages仓库：

  1. 访问 GitHub 并登录
  2. 创建新仓库，命名为 4ier.github.io
  3. 确保仓库是 公开的
  4. 勾选 "Add a README file"
2 克隆仓库到本地
# 克隆你的GitHub Pages仓库 git clone https://github.com/4ier/4ier.github.io.git cd 4ier.github.io
3 复制文档站点文件

将docs-site目录中的所有文件复制到你的GitHub Pages仓库：

# 复制所有文档文件（假设retire-cluster和4ier.github.io在同一目录下） cp -r ../retire-cluster/docs-site/* ./ # 或者使用绝对路径 cp -r /path/to/retire-cluster/docs-site/* ./
4 配置GitHub Pages
  1. 在GitHub上打开你的仓库设置
  2. 在侧边栏找到 Pages 部分
  3. 在 Source 下选择 GitHub Actions
  4. 保存设置
5 提交和推送代码
# 添加所有文件 git add . # 创建提交 git commit -m "部署Retire-Cluster文档站点 - 完整的Retire-Cluster项目文档 - 双语支持（中英文） - 响应式设计和移动端优化 - 自动GitHub Actions部署 - 专业样式和导航 🚀 在线访问: https://4ier.github.io" # 推送到GitHub git push origin main
🎉 部署完成！
GitHub Actions将自动构建和部署你的站点。几分钟后，你的文档将在 https://4ier.github.io 上线。

💡 提示： 我们为你准备了一个自动化部署脚本 deploy.sh，可以自动完成上述所有步骤。
# 使用自动化脚本部署 cd /path/to/retire-cluster/docs-site chmod +x deploy.sh ./deploy.sh ../4ier.github.io

🔧 自定义和维护

添加新项目

要在主页添加新的开源项目，编辑 index.html 文件，在项目网格中添加新的项目卡片。

更新文档

要更新Retire-Cluster文档，只需：

  1. 在retire-cluster仓库中更新Markdown文件
  2. 重新运行部署脚本
  3. GitHub Actions会自动将Markdown转换为HTML并部署

样式定制

站点使用CSS自定义属性，便于主题定制：

:root { --primary-color: #007acc; /* 主色调 */ --secondary-color: #005a99; /* 辅助色 */ --bg-color: #ffffff; /* 背景色 */ --text-color: #333333; /* 文字颜色 */ }
⚠️ 注意： 第一次部署后，GitHub Pages可能需要几分钟才能生效。如果遇到404错误，请稍等片刻再次访问。
